2. S01 — Fetch 自动化

目的：调用 NOX Influencer API，按平台获取每个 influencer 在指定日期范围内发布的视频。

输入 / 输出

类型文件 / 内容
输入Influencer 列表（按平台分组），参数 start_date、end_date
输出s01_fetch/all_videos.tsv — 视频字段
输出s01_fetch/all_fetch_errors.tsv — 失败记录
输出s01_fetch/all_raw_nox.jsonl — NOX API 原始 JSON

输出列

inf_id, alias, platform, channelId, videoId, url, publishTime,
views, likes, comments, tags, title, description

NOX API 端点

平台URL
YouTubehttps://service.noxinfluencer.com/nox/v1/youtube/video
TikTokhttps://service.noxinfluencer.com/nox/v1/tiktok/video
Instagramhttps://service.noxinfluencer.com/nox/v1/instagram/video
配额查询https://service.noxinfluencer.com/nox/v1/key?noxKey={KEY}

请求参数

{
  "noxKey": "<API_KEY>",
  "channelId": "<channel_id>",
  "startDate": "2026-01-01",
  "endDate":   "2026-01-07",
  "country":   "US",
  "language":  "en",
  "pageSize":  100,
  "page":      1
}

country / language 为 NOX 地区与语言参数，来自配置 NOX_REGION、NOX_LANGUAGE。

分页与重试

  • 从 page=1 开始，每页 100 条，直到 len(items) >= total
  • 成功条件：api_code == 10000
  • 可重试错误码：{50001, 50002, 50003}
  • 最大重试 3 次，间隔 2 秒
调度策略：平台轮转（round-robin）均衡请求；每批 10 个 influencer 用 ThreadPoolExecutor 并行。 标题/描述中的换行、Tab 被替换为 ||（TSV 安全）。

User Prompt — 完整原文

以下是发送给 LLM 的 User Prompt 完整模板（来自 lib/product_extractor.py 的 USER_PROMPT_TEMPLATE，占位符 {content}、{category_context}、{global_exclusions}、{current_date} 在运行时替换）：

展开 / 折叠 User Prompt 完整原文 
You are a product extraction expert. Extract products mentioned in social media posts.

Current date: {current_date}
Content:
{content}
{category_context}
{global_exclusions}

HARD RULES (must always be followed):

1. The product name MUST appear explicitly in the content text.
   - Do NOT guess or complete product names from your knowledge.
   - EXCEPTION — Collaborations: If the text clearly describes a collaboration (e.g. "X has reunited with Y to put his spin on Z",
     "X's collaboration with Y", "X x Y"), you MUST construct the product name as "CollabPartner x ProductLine"
     (e.g. "@lorenz.og ... his latest spin on the Air Max 95" → name="Lorenzo x Air Max 95").
     Both the collab partner AND the product line must appear in the text.
   - CO-BRAND PRESERVATION (HARD): If the text explicitly ties two brands/entities to the SAME product
     (e.g. "Crocs and LEGO ... Brick Clog", "LEGO Crocs Brick Clog", "BrandA x BrandB ProductLine"),
     the extracted product name MUST retain both entities (e.g. "LEGO x Crocs Brick Clog"), not a single-brand rewrite.
     Do NOT drop, replace, or normalize away an explicitly mentioned collaborator token.

2. The brand MUST be identifiable from the text (via @mention, explicit name, or hashtag).
   - Retailers (Ulta, Sephora, Amazon, Target) are NOT brands.
   - Use the OFFICIAL brand name.

3. For each product, provide "evidence": copy the EXACT substring from the original text that contains the product name.

4. For each product, judge its RELEVANCE for homepage display:
   - "on_topic": product is a real, purchasable consumer product suitable for a shopping homepage
   - "off_topic": product falls under GLOBAL EXCLUSIONS (vehicles, movies, toys, swimwear, subscription services, etc.),
     OR is a promotional freebie / giveaway item that CANNOT be purchased separately
     (e.g. McDonald's collectible cards, Happy Meal toys, free gifts with purchase, loyalty rewards)
     → Subscription services (Netflix, Game Pass, Spotify, Disney+, etc.) are ALWAYS off_topic — never extract
   - "borderline": product is unusual but still a legitimate purchasable item
   Only extract products that are "on_topic" or "borderline". Do NOT extract "off_topic" products.

HOMEPAGE INCLUSION CRITERIA (MUST satisfy all):
1) Purchasable: The item is a real consumer product/service that can be obtained independently.
2) Searchable: The text provides enough identity to search it (at least brand + product name/type).
3) Primary focus: The product is a main subject of the post (review/recommendation/demo), not incidental tags.

If any criterion is not satisfied, do NOT include in products, regardless of confidence.
Campaign/engagement slogans without concrete product evidence (e.g. #JoinTeamX, #XChallenge, #Giveaway) must be capped at 2.

5. PRODUCT NAME COMPLETENESS: If the product name is only a single word (e.g. "Rapunzel", "Elixir"),
   append its product type/category to make it a searchable name.
   - WRONG: name="Rapunzel", category="fragrance"  (searching "Disney Rapunzel" finds a movie character)
   - RIGHT: name="Rapunzel Fragrance", category="fragrance"  (searching "Disney Rapunzel Fragrance" finds the product)
   - Multi-word names like "Soft Pinch Liquid Blush" or "Airwrap Styler" are already specific enough — do NOT add category to them.
   VARIANT / SHADE / COLORWAY INCLUSION (HARD RULE):
   When the text specifies a shade, color, variant, size, or edition (e.g. "in 09 dreamy Dawn", "shade 35",
   "color Opal", "size 8", "Limited Edition"), you MUST include it in the product name.
   - WRONG: name="Cloud Topia Blush"  (shade "09 dreamy Dawn" is mentioned but lost)
   - RIGHT: name="Cloud Topia Blush 09 Dreamy Dawn"
   - WRONG: name="Lifter Concealer"  (shade 35 is mentioned but lost)
   - RIGHT: name="Lifter Concealer Shade 35"
   This ensures downstream search queries can match the exact variant the creator used.

6. LANGUAGE CHECK (MUST be done FIRST, before any extraction):
   Determine the PRIMARY language of the post content. Ignore brand names, product model numbers, and hashtags when judging language.
   If the primary language is NOT English, immediately return empty products with failType="non_english".
   Do NOT extract products even if the brand or product name appears to be in English.

SPAM / SEO TAG DETECTION (MUST be checked BEFORE scoring candidates):
Before scoring any candidates, compare the post TITLE to the products you found.
If the title clearly indicates the post is about a SPECIFIC TOPIC (e.g. a game like Roblox/Fortnite/Free Fire,
a challenge, a movie reaction, a comedy skit) and the product mentions appear ONLY in description hashtags
or keyword-stuffed text that is UNRELATED to that topic, those are SEO spam tags — NOT genuine product mentions.
→ Cap ALL such spam-tag products at confidence 2, regardless of how specific the model name looks.
Example: Title="Rescuing My Roblox Friends from Golden Tsunami"
  Description has "a01,a11,a10,a20,a30,...,iphone, #Iphone8plus, #j5prime, #j2prime"
  → The post is about Roblox gameplay; these phone models are spam SEO tags → score 1-2 for ALL of them.
Only apply this when the mismatch is CLEAR. If the title is vague or the products plausibly relate, score normally.

VIDEO-CONTENT FILTERING (MUST be checked BEFORE scoring candidates):
If the post is primarily video-first entertainment or creator content (e.g. gameplay, prank, reaction, dance,
comedy skit, podcast clip, sermon/motivation clip, sports highlight, meme/viral edit) and product words are only
incidental mentions (hashtags, ambient background, creator handle, generic "shop link in bio"), do NOT treat them
as homepage-ready products.
→ Such incidental mentions should usually be capped at confidence 0-2 and not included in products.
→ Only allow confidence >= 5 when the post clearly foregrounds a purchasable product as a main subject with explicit
  identity evidence (brand + product name/type + concrete mention in title/description).
→ For these content-first posts (including tutorial/how-to explainers) where commerce is NOT the primary intent,
  set failType="non_commercial".

GAMEPLAY-SPECIFIC HEURISTICS:
- If title/caption is clearly gameplay-centric (e.g. "ranked match", "clutch", "speedrun", "boss fight", "live gameplay"),
  treat game names, skins, map names, mods, and in-game items as non-shopping content by default.
- Device/model names that appear only in hashtag blocks or SEO stuffing (e.g. phone model lists under gameplay clips)
  should be treated as spam-like incidental mentions and capped at 0-2.
- Only keep confidence >= 5 when there is explicit real-world product intent (e.g. "I use X mouse", "buy this headset",
  "setup details: brand + product") with concrete purchasable evidence in the post text.

ASMR-SPECIFIC HEURISTICS:
- If the post is primarily ASMR/performance content (trigger words like "asmr", "satisfying", "sleep sounds", "no talking"),
  do NOT extract incidental objects/tools used for sound-making as products unless they are explicitly recommended/sold.
- Generic object mentions in ASMR context (soap, slime, brushes, tapping tools, cups, knives, etc.) are usually props,
  not purchasable product recommendations; cap at 0-2 unless explicit shopping intent exists.
- For ASMR creator promos, require strong evidence (brand + concrete product name + purchase/recommendation language)
  before allowing confidence >= 5.

When choosing failType for content-first posts:
- Use "non_commercial" for content-first video (gameplay/ASMR/tutorial/reaction/performance) with incidental or weak product signals
- Use "no_product" ONLY when there is truly no product-like mention at all
- Use "lifestyle_only" ONLY for pure lifestyle scenes (no commercial intent, no brand/product reference)
- Use "music_dance" ONLY for clear music/dance-only clips

CANDIDATE SCORING:
List ALL product-like mentions in the text as candidates, even if you are unsure.
The confidence score (0-9) determines HOW SPECIFIC the product identification is — this replaces any yes/no extraction rule.
Score based on how clearly the text identifies a product — NOT on whether you recognize the product.
You cannot search the web. Do NOT penalize a product just because you are unfamiliar with it.
If the text signals "this is a specific product" (e.g. hashtag as a name, brand handle + product term, recommendation language like "must have"), trust the text.

Score each candidate's confidence from 0 to 9:
  9: Specific product name + colorway/edition/variant + brand clearly stated
     (e.g. "Air Jordan 1 High Alaska", "Soft Pinch Liquid Blush in Joy")
  8: Product is clearly specific and searchable (brand + distinct product name),
    but missing variant-level detail (e.g. no explicit colorway/shade/size/capacity/edition).
     Typical 8 examples:
       - Brand + distinct product name that reads near-unique in context
       - Specific collab/product name that is clearly formed and directly searchable
  7: Product is specific with brand identifiable (often model/version/collab/distinct name),
    but some naming context is still incomplete or slightly ambiguous.
     Typical 7 examples:
       - Names with model/version numbers (e.g. "EP-133 sampler", "ABZORB 2010", "991v1", "990v4")
       - A collaboration/collab version of a product line (e.g. "Lorenzo x Air Max 95", "Travis Scott x Air Jordan 1")
         → The collab name MUST appear in the extracted product name (e.g. name="Lorenzo x Air Max 95", NOT just "Air Max 95")
       - Co-branded names where BOTH entities are explicit in text (e.g. "LEGO x Crocs Brick Clog")
         → Dropping one co-brand entity reduces specificity and should be scored lower.
       - A distinct hashtag that clearly functions as a product name (e.g. "#AutoScooper")
         → EXCLUDE campaign/engagement slogans (e.g. "#JoinTeamX", "#XChallenge", "#Giveaway", "#Contest") unless concrete product evidence exists
       - Post is clearly recommending/reviewing ONE product and you can identify a name + brand → usually 7+
  6: A product line or named range WITHOUT a model number (e.g. "Air Max 95", "Air Max 90", "KERATIME" — product series/lines),
     a collab product where the collab partner is NOT captured (just "Air Max 95" without the collab name → stays at 6),
     note: series numbers like "95", "90", "97" identify which SERIES, not a specific variant — they are NOT model/version numbers,
     or brand + MODIFIED PRODUCT SUBTYPE — the type word carries a qualifying modifier
     (material / function / form-factor / usage) that narrows it to a recognizable sub-category.
     TEST: does the type word contain a modifier that limits the scope to a specific sub-segment of the base category?
     If yes → 6. If the type word is a bare category with no modifier → 5.
     ✅ 6 examples: "running shoes" (function), "cushion foundation" (form), "matte lipstick" (finish),
       "crossbody bag" (form), "wireless earbuds" (tech), "zip-up hoodie" (form), "bomber jacket" (style),
       "lip gloss" (already a distinct sub-category of lip products)
     ❌ stays at 5: "sneakers", "shoes", "lipstick", "foundation", "bag", "hoodie", "jacket"
  5: Brand + a BARE product category/type with NO qualifying modifier.
     The category is still a recognizable, usable product type, but it maps to a large number of products.
     Examples: "Nike sneakers" (bare category), "Adidas hoodie" (bare category), "Dior lipstick" (bare category)
     CRITICAL: The product category/type word (e.g. "sneakers", "hoodie", "lipstick", "floor cleaner")
     MUST appear explicitly in the text. You CANNOT infer the category from your knowledge of the brand.
     If only the brand name appears with NO product type word in the text, this is a brand-only mention → score 3.
     Example: "Narwal's Spring Sale is LIVE" → only brand "Narwal" appears, no product type word → score 3, NOT 5.
     EXCEPTION: If the brand is a niche/single-product brand where "brand + bare category" effectively identifies
     ONE specific product (e.g. a brand that only makes one lipstick), score 6 instead — it narrows to a small set like a modified subtype.
  4: Brand + MIXED collection / mixed-category product bucket (e.g. "new arrival", "spring collection", "limited edition", "trending", "merch", "capsule", "drop", "kit").
    This refers to a collection that can contain multiple product types rather than one stable product category.
    Examples: "Nike new arrival" (mixed collection), "Adidas spring collection" (mixed types)
  3: Brand OR category mentioned alone (not both together), or both present but too vague to form a searchable query.
     Examples: just "@nike" with no product term, just "sneakers" with no brand, "check out their stuff"
  2: Weak but explicit product-like signal exists (e.g. campaign/engagement slogan or weak brand+term),
    but still not concretely searchable as a product.
  1: Seems product-like but uncertain; confirming it would require guessing.
  0: Not a product at all (e.g. a person's name, a movie title, a location)

KEY scoring rules:
- Model/version numbers (e.g. "2010", "v1", "v2", "133") → score 7+
- Collaboration name included in product name (e.g. "Lorenzo x Air Max 95") → score 7+
- Hashtag can be score 7+ ONLY when it clearly denotes a product name (e.g. "#AutoScooper")
- Campaign/engagement slogans (e.g. "#JoinTeamX", "#XChallenge", "#Giveaway", "#Contest") with no concrete product evidence → cap at 2
- Product series/line (e.g. "Air Max 95", "Air Max 90", "KERATIME") → 6 (series numbers like 95/90/97 are NOT model/version numbers)
- Collab product WITHOUT the collab name captured → 6 (missing the specificity)
- Brand + MODIFIED product subtype (type word has a qualifying modifier: material/function/form/usage) → 6
  (e.g. "running shoes", "cushion foundation", "matte lipstick", "crossbody bag", "lip gloss")
- Brand + BARE product category (no modifier, e.g. "sneakers", "hoodie", "lipstick", "bag") → 5
- Brand + product type listed/mentioned without strong endorsement → 5
- Mixed collection / mixed-category bucket (new arrival, spring collection, holiday collection, trending picks, kit) → 4
- Generic collection words like "merch", "capsule", "drop", "essentials" are 4 by default (mixed bucket)
  → Only treat as 5 when the post explicitly narrows to a single product type (e.g. "brand tee", "brand hoodie")
- Broad bucket words such as "gear", "apparel", "products", "cosmetics", "makeup", "skincare", "merch", "kit" are 4 by default
  → Only allow 5 when explicitly narrowed to ONE stable item type (e.g. "running jacket", "matte lipstick", "football jersey")
- Brand-only or handle-only mention with no concrete item type (e.g. just "@brand" / brand name) → 3 (never 5)
- Category-only word with no brand (e.g. just "sneakers" with no brand) → 3
- Pure category word without enough clear product identity → 3-4
- Implicit promotion (e.g. "@brand product" in a list or lifestyle post without explicit endorsement language like "recommend", "must have", "love", "review") → cap at 5
  → Only @brand + product name with NO endorsement signals → score 5 or lower
- If uncertain between adjacent scores, choose the LOWER score.

CRITICAL RULE: After scoring all candidates, you MUST populate "products" with EVERY candidate whose confidence >= 5
that is "on_topic" or "borderline" AND satisfies all HOMEPAGE INCLUSION CRITERIA.
This is a hard rule — do NOT override it based on subjective judgment. If a candidate meets these conditions, it goes into products.

Only candidates with confidence >= 5 that also pass relevance + homepage gates go into "products". All candidates (including rejected ones) go into "candidates".

Output JSON only:
{
  "candidates": [
    {
      "name": "best product name you can identify from text",
      "brand": "brand from text",
      "confidence": 8,
      "reason": "why this score — what makes it specific or vague"
    }
  ],
  "products": [
    {
      "name": "official product name from text",
      "brand": "brand from text",
      "confidence": 8,
      "category": "product category",
      "evidence": "exact quote from text",
      "relevance": "on_topic",
      "relevance_reason": "brief reason"
    }
  ],
  "failType": ""
}

failType options (when no products with confidence >= 5):
- "no_product": No product or brand mentioned at all
- "lifestyle_only": Pure lifestyle content
- "music_dance": Music/dance/entertainment
- "empty_content": Too short to analyze
- "no_spec_prod": Brand mentioned but no product name reached confidence >= 5
- "non_commercial": Non-commercial-intent content — educational, news, gameplay, ASMR, reaction/prank/comedy video, tutorial, performance, or any content where commerce is NOT the primary purpose
- "restaurant_food": Restaurant/venue items only
- "off_topic": Products found but all are outside the vertical category
- "secondhand_outdated": Products found but all appear secondhand or outdated
- "non_english": Content is not in English

- "candidates" contains ALL product-like mentions regardless of confidence

- If products is non-empty, failType=""
- If products is empty, set appropriate failType
- Output JSON only, no other text

User Prompt — 中文摘要（便于理解）

规则 1 — 产品名必须出现在文本中

  • 不得猜测或补全产品名
  • 合作款例外：文本明确描述合作关系 → 格式 CollabPartner x ProductLine（如 "Lorenzo x Air Max 95"）
  • 联名品牌保留：文本中明确关联双品牌 → 提取名称必须保留双方（如 "LEGO x Crocs Brick Clog"）

规则 2 — 品牌必须可识别

  • 通过 @提及、显式名称或 hashtag 识别
  • 零售商（Ulta, Sephora, Amazon, Target）不算品牌
  • 使用官方品牌名

规则 3 — Evidence

复制原文中包含产品名的原文子串作为证据。

规则 4 — 首页展示相关性判断

判定说明
on_topic真实可购买的消费品，适合购物首页
off_topic全局排除类别 / 促销赠品（不可单独购买）
borderline不常见但仍为合法可购买商品
全局排除（off_topic）：Vehicles · Movies · Toys · Swimwear · 订阅服务（Netflix / Game Pass / Spotify / Disney+）· 促销赠品（McDonald's 收藏卡、快乐儿童餐玩具、购买赠品、会员奖品）

首页准入三条件（全部满足才提取）

  1. Purchasable：真实消费品/服务，可独立获取
  2. Searchable：至少 brand + product name/type
  3. Primary focus：产品是帖子主题（评测/推荐/演示），非附带标签

规则 5 — 产品名完整性

  • 单字产品名须追加类型（"Rapunzel" → "Rapunzel Fragrance"）
  • 色号/变体硬规则：文本中提到的色号、颜色、变体、尺寸、版本必须包含在产品名中
    ❌ "Cloud Topia Blush"  →  ✅ "Cloud Topia Blush 09 Dreamy Dawn"

规则 6 — 语言检查（最先执行）

非英语 → 直接返回空产品，failType="non_english"（忽略品牌名和 hashtag 的语言）

置信度评分体系（0-9）

分数定义典型示例
9具体产品名 + 色号/版本/变体 + 品牌明确"Air Jordan 1 High Alaska"、"Soft Pinch Liquid Blush in Joy"
8品牌 + 独特产品名，但缺变体级信息品牌 + 独特产品名，可直接搜索
7型号/版本/合作款名称"EP-133 sampler"、"Lorenzo x Air Max 95"、"991v1"
6产品线/系列（无型号），或品牌 + 带修饰词的产品子类型"Air Max 95"（系列号≠型号）、"running shoes"、"matte lipstick"
5品牌 + 裸产品类别（无修饰词）"Nike sneakers"、"Dior lipstick"
4品牌 + 混合集合/多品类概念"new arrival"、"spring collection"、"merch"、"capsule"
3仅品牌或仅类别单独出现仅 "@nike" / 仅 "sneakers"
2弱但显式的产品信号活动口号、弱品牌+术语
1疑似产品但不确定需猜测才能确认
0非产品人名、电影名、地点
提取阈值：confidence >= 5 且 relevance 为 on_topic/borderline 且满足首页准入条件 → 进入 products。
关键规则：相邻分数犹豫时选低分；hashtag 达 7+ 仅当它明确代表产品名（如 #AutoScooper）；活动口号（#JoinTeamX, #Giveaway）上限 2。

内容过滤规则

内容类型处理
SEO 垃圾标签标题主题与描述 hashtag 中产品名不匹配 → 全部上限 2
游戏内容游戏名/皮肤/地图/模组/游戏内物品 → 非购物，上限 0-2
ASMR 内容通用道具（肥皂、软泥、刷子）→ 上限 0-2，除非有明确购物意图
视频优先内容游戏/整蛊/反应/舞蹈/喜剧/播客 → failType="non_commercial"

failType 枚举

failType含义
no_product无任何产品/品牌提及
lifestyle_only纯生活方式内容
music_dance音乐/舞蹈/娱乐
empty_content内容太短无法分析
no_spec_prod有品牌但无产品名达到 confidence >= 5
non_commercial非商业意图（教育/新闻/游戏/ASMR/反应等）
restaurant_food仅餐厅/场地项目
off_topic产品属于排除品类
secondhand_outdated二手/过时产品
non_english非英语内容

输出 JSON 格式

{
  "candidates": [
    {"name": "...", "brand": "...", "confidence": 8, "reason": "..."}
  ],
  "products": [
    {
      "name": "...", "brand": "...", "confidence": 8,
      "category": "...", "evidence": "...",
      "relevance": "on_topic", "relevance_reason": "..."
    }
  ],
  "failType": ""
}

4. S03 — Dedup 自动化

目的：两轮去重 —— 先字符串精确匹配，再用 LLM 语义去重。

输入 / 输出

输入s02_extract/all_products.tsv
输出s03_dedup/all_products.tsv — 去重后唯一产品
输出s03_dedup/all_dedup_removed.tsv — 被去除的重复项及映射

去重流程

1. 字符串去重 — 按 brand+productName 精确分组
       ↓
2. 分配 UUID — 每行获得 8 字符 hex _dedup_id
       ↓
3. LLM 去重 — 产品数 >= 2 时触发
       ↓
4. 孤儿处理 — 不在 kept/removed 中的 ID 默认保留
       ↓
5. 审计构建 — 记录 removed→kept 映射

System Prompt

You are a specialized Product Data Engineer.
Your task is to deduplicate product lists based on
specific category-level business rules.
Output only valid JSON.

品类级合并规则

品类SAME（合并）DIFFERENT（保留）
美妆护肤同色号不同命名（"Dior 999" vs "Dior Rouge 999"）不同色号（"Dior 999 Red" vs "Dior 100 Nude"）

户外运动—不同颜色（夹克/绳索）、不同材质（帐篷）、不同温标（睡袋）

通用清理规则

  • 合并大小写、缩写、拼写错误变体
  • 移除修饰词："Best Seller"、"New"、"Official"

输出 JSON 格式

SerpAPI 请求

Step 1：Google Shopping Light 搜索

{
  "q":        "{brand} {productName}",
  "api_key":  "<SERP_API_KEY>",
  "engine":   "google_shopping_light",
  "gl":       "us",
  "hl":       "en"
}

Step 2：Immersive Product 详情页

{
  "engine":     "google_immersive_product",
  "page_token": "<from shopping_results[].immersive_product_page_token>",

匹配逻辑

  5. 按商家名去重（不区分大小写）
  6. 返回前 N 个结果（默认 SERP_TOP_N=3）

参数默认值

System Prompt

You are a quality-verification assistant for an influencer product pipeline.

Score each catalog product against the original social post on a 0-5 scale.

Scoring:
  5: Brand + product + variant all align.
  4: Brand + product line align, variant differs.
  3: Brand + category align, specific line/model is generic or unclear in post.
  2: Brand/category near-match but specific line/model does not match.
  1: Brand only, category differs.
  0: Unrelated or negative sentiment mention.

Return ONLY JSON:
{
  "results": [

  ]
