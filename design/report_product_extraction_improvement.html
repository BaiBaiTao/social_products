Prompt 改进内容

Product Extraction prompt 经历了 v2 和 v2.1 两次迭代，核心改进：

改进点改进前问题改进后方案
1. 结构化 PRODUCT NAME CONSTRUCTION Rule 1 仅要求"名称必须出现在文本中"，LLM 经常漏抓变体/材质/联名等信息，或重复输出 定义 7 个 slot (collab/collection/product_line/model/variant/material/edition)，按固定顺序拼接名称。每个 token 必须来自文本，不能靠知识补全
2. Material Slot 强化 v2.1 v2 中 material 定义为"显式描述符"，#1 AF1 漏抓 "Suede" 改为"区分 SKU 的材质"：同一鞋款 Suede vs Leather = 不同 SKU → 必须提取。即使出现在 "suede upper" 等短语中也要捕获
3. Cross-Sentence Variant Association 配色/色号出现在不同句子时被忽略（如 "2X Ultra Cover" 在前句、"Poppy Red" 在后句） 新增规则：当帖子明确推广一个产品且色号/变体出现在别处指代该产品时，跨句关联
4. Collection Slot 新增 系列名/collection（如 "Mod Collection 2025"、"Color Watch 2026"）无处安放 新增 collection slot，放在 collab 和 product_line 之间，解决系列名缺失
5. Title-Anchored Main Subject 多型号帖子中所有提及的同品牌型号都被提取（如 ASUS Zenbook A16 + A14） 当标题明确命名一个产品时，仅提取该主角产品；同品牌其他型号仅为辅助提及不提取
6. Comparison Rule 对比视频中 A vs B 两个产品都被提取 仅提取被推荐/评测的主角，对比基线产品排除
7. 品牌名去重 (Brand-in-Name) 输出 "CaIpak - CaIpak Hue Carry-On"，品牌重复 规则明确 "Do NOT repeat brand in name"
8. Evidence 放松 旧版要求 evidence 包含完整产品名的连续引用，结构化名字各部分散布时无法满足 允许引用多个不连续文本片段，只需覆盖各 slot 的原始文本
9. Variant/Shade 强制包含 文本明确提到色号/变体但被忽略（如 "shade 35"、"09 Dreamy Dawn"） VARIANT / SHADE / COLORWAY INCLUSION (HARD RULE)：明确提到的必须包含在名称中
10. Product Name Completeness 单词产品名（如 "Rapunzel"）搜索时匹配到无关内容 单词名称必须追加 category 使搜索可行（如 "Rapunzel Fragrance"）
