Stage Details & LLM Prompts

S01
NOX Video Fetch
Pull recent videos from influencer accounts via NOX Influencer API
No LLM ▾

Input

  • Influencer list TSV (config/influencers*.tsv)
  • Date range (daily, default today)
  • --limit N influencers (round-robin across platforms)

Output

  • s01_fetch/all_videos.tsv — structured video rows
  • s01_fetch/all_raw_nox.jsonl — full raw API responses
  • s01_fetch/errors.tsv — API failures

Platforms

  • YouTube
  • TikTok
  • Instagram

Columns Output

inf_id, alias, platform, channelId, videoId, url, publishTime, views, likes, comments, tags, title, description

S02
LLM Product Extraction
Identify specific shoppable products from video title + description
🤖 LLM ▾

Input

  • s01_fetch/all_videos.tsv
  • Video title + description (combined, max 800 chars)

Output

  • s02_extract/all_extracted.tsv — all rows (products + fails)
  • Rows with brand + productName = extracted products
  • Rows with failType set = failed videos:
    no_product / non_english / lifestyle_only /
    music_dance / no_spec_prod / off_topic / etc.

Failure Handling

  • no_product: nothing mentioned
  • non_english: non-English content → skip
  • lifestyle_only, music_dance, etc.
  • Max 3 retries per video, 30s timeout

Design Principles

  • Never guess product names — must be in text
  • Brand must be @mentioned or explicit
  • Single-word names get type appended for searchability
  • Global exclusion list rejects off-topic products

LLM Prompts (Azure OpenAI)

System 
You are a strict product extraction expert. Output only valid JSON. Never guess product names.
User 
You are a strict product extraction expert. Extract ONLY products whose SPECIFIC OFFICIAL NAME appears in the text below.

Current date: {current_date}
Content:
{title}
{description}
{global_exclusions}

STRICT RULES:

1. PRODUCT NAME must UNIQUELY identify ONE specific purchasable product, not a category.
   - KEY TEST: if you search "brand + name" on the brand's website, would it return ONE specific product page
     with a price? If yes, extract. If it returns a category page listing multiple products, do NOT extract.
   - WRONG (category descriptions): "foundation", "mascara", "blush", "primer"
   - RIGHT (unique product names): "Soft Pinch Liquid Blush", "Shape Tape Concealer", "Airwrap"

2. The product name MUST appear explicitly in the content text.
   - Do NOT guess or complete product names from your knowledge.

3. The brand MUST be identifiable from the text (via @mention, explicit name, or hashtag).
   - Retailers (Ulta, Sephora, Amazon, Target) are NOT brands.
   - Use the OFFICIAL brand name.

4. When in doubt, do NOT extract. Better to miss than to extract wrong.

5. For each product, provide "evidence": copy the EXACT substring from the original text that contains the product name.

6. For each product, judge its RELEVANCE for homepage display:
   - "on_topic": product is a real, purchasable consumer product suitable for a shopping homepage
   - "off_topic": product falls under GLOBAL EXCLUSIONS (vehicles, movies, toys, swimwear, etc.)
   - "borderline": product is unusual but still a legitimate purchasable item
   Only extract products that are "on_topic" or "borderline". Do NOT extract "off_topic" products.

7. PRODUCT NAME COMPLETENESS: If the product name is only a single word (e.g. "Rapunzel", "Elixir"),
   append its product type/category to make it a searchable name.
   - WRONG: name="Rapunzel", category="fragrance"  → searching "Disney Rapunzel" finds a movie character
   - RIGHT:  name="Rapunzel Fragrance", category="fragrance"

8. LANGUAGE CHECK (MUST be done FIRST):
   If the primary language is NOT English → return failType="non_english" immediately.

Output JSON only:
{
  "products": [
    {
      "name": "official product name from text",
      "brand": "brand from text",
      "category": "product category",
      "evidence": "exact quote from text",
      "relevance": "on_topic",
      "relevance_reason": "brief reason"
    }
  ],
  "otherInfo": "",
  "failType": ""
}

failType options: no_product | lifestyle_only | music_dance | empty_content | no_spec_prod
                  non_commercial | restaurant_food | off_topic | secondhand_outdated | non_english
S03
Deduplication
Two-pass dedup: fast string similarity then LLM semantic merge
🤖 LLM ▾

Pass 1 — String Dedup

  • Normalise: lowercase, strip punctuation
  • Group by exact normalised (brand, productName)
  • Fast, no API cost

Pass 2 — LLM Dedup

  • Category-aware merging rules
  • Beauty: different shades = DIFFERENT
  • Electronics: different colors = SAME
  • Apparel: different sizes = SAME

Input

  • s02_extract/all_extracted.tsv (only rows with brand + productName, i.e. failType="")

Output

  • s03_dedup/all_products.tsv — unique canonical products
  • s03_dedup/all_dedup_removed.tsv — merged duplicates log

LLM Prompts (Azure OpenAI)

System 
You are a specialized Product Data Engineer. Your task is to deduplicate product lists based on specific category-level business rules. Output only valid JSON.
User 
Below is a list of (brand, productName) pairs. Your goal is to identify DUPLICATE items and merge them according to strict category definitions.

### Category Definition & Merging Rules:

1. **Beauty & Fragrance** (makeup, skincare, perfume, haircare):
   - DIFFERENT: Different shades/colors → DIFFERENT ("Dior 999 Red" ≠ "Dior 100 Nude").
   - DIFFERENT: Different scents → DIFFERENT ("Jo Malone Peony & Blush Suede" ≠ "Jo Malone Wood Sage & Sea Salt").
   - SAME: Same shade/scent with minor naming variation → merge ("Dior 999" = "Dior Rouge 999").

2. **Electronics / Computing / Cameras & Optics**:
   - SAME: Different colors → SAME, strip color ("Sony WH-1000XM5 Black" + "Silver" → "Sony WH-1000XM5").
   - DIFFERENT: Different models or generations → DIFFERENT ("GoPro Hero 12" ≠ "GoPro Hero 13").
   - DIFFERENT: Different storage/RAM capacities → DIFFERENT.
   - DIFFERENT: Different lens focal lengths or sensor sizes → DIFFERENT.

3. **Shoes**:
   - SAME: Different sizes → SAME, strip size.
   - DIFFERENT: Different colors or colorways → DIFFERENT.
   - DIFFERENT: Different models → DIFFERENT.

4. **Sports & Outdoors**:
   - DIFFERENT: Different colors of apparel/jackets → DIFFERENT (visibility/safety matters).
   - DIFFERENT: Different temperature ratings for sleeping bags → DIFFERENT.
   - DIFFERENT: Different materials or specs for tents/ropes/helmets → DIFFERENT.

5. **Home Furnishings / Kitchen & Housewares**:
   - DIFFERENT: Different colors or finishes → DIFFERENT.
   - DIFFERENT: Different scents for candles or cleaning products → DIFFERENT.
   - SAME: Same product, only minor size variation, no functional difference → merge.

6. **Lawn & Garden**:
   - DIFFERENT: Different volumes or concentrations → DIFFERENT.
   - DIFFERENT: Different plant types or formulations → DIFFERENT.

7. **Toys**:
   - DIFFERENT: Different characters, themes, or sets → DIFFERENT.

8. **Car & Garage**:
   - DIFFERENT: Different fitments, sizes, or vehicle compatibility → DIFFERENT.

### General Cleaning Rules:
- Merge casing/abbreviation/typo variations ("MAC Eye Shadow" = "MAC Eyeshadow").
- Remove fluff words: "Best Seller", "New", "Official".
- Strip color/size suffix ONLY when category rules above say SAME.

Products to Process:
{product_list}

Output Format:
{
  "unique": [
    {"brand": "canonical brand", "productName": "canonical product name"}
  ],
  "duplicates": [
    {
      "kept": {"brand": "...", "productName": "..."},
      "removed": [{"brand": "...", "productName": "..."}]
    }
  ]
}
S04
L1 Category Classification & Filter
Assign standardised L1 taxonomy tag; drop excluded categories before SERP
🤖 LLM ▾

Purpose

  • Classify each product into one of 31 standard L1 categories
  • Filter out 20 excluded categories before SERP — saves API quota
  • Outputs LLM_L1Category column for downstream use

11 Kept Categories

  • Home Furnishings, Kitchen & Housewares
  • Electronics, Computing, Cameras & Optics
  • Sports & Outdoors, Lawn & Garden
  • Shoes, Beauty & Fragrance
  • Toys, Car & Garage

Input

  • s03_dedup/all_products.tsv
  • config/l1_tags.tsv — 31 L1 label names
  • config/l1_exclude_tags.txt — 20 excluded labels

Output

  • s04_l1/all_l1.tsv — kept rows + LLM_L1Category
  • s04_l1/all_l1_excluded.tsv — excluded rows for review

S05
SERP Search & LLM Verification
Google Shopping search via SerpApi + LLM validates each result is the correct product
🤖 LLM ▾

Search Phase (SerpApi)

  • Query: "brand productName" on Google Shopping
  • Up to 3 results per product
  • 20 parallel workers
  • Fallback: Immersive Shopping results

Verify Phase (LLM)

  • LLM checks each SERP result vs extracted product
  • Must match: brand, product type, specific item
  • Color/size variants OK; different categories → NO
  • Accessories for the product → NO

Input

  • s04_l1/all_l1.tsv

Output

  • s05_serp/all_serp_matched.tsv — verified matches
  • s05_serp/all_serp_mismatch.tsv — LLM-rejected
  • s05_serp/all_no_serp.tsv — no Google results

LLM Prompts — SERP Verification (Azure OpenAI)

System 
You are a strict product matching expert. You determine whether a search result refers to the SAME product the user is looking for.
User 
A product was extracted from a video:
  Brand:    GoPro
  Product:  GoPro Hero 12
  Category: Cameras & Optics

Below are Google Shopping search results returned for this product.
For EACH result, determine: is it the SAME product (same brand, same specific item, same category)?

Search results:
  [1] Title: GoPro HERO12 Black — Waterproof Action Camera  |  Price: $299.99  |  Source: gopro.com
      GoPro HERO12 Black Action Camera including...
  [2] Title: GoPro HERO 12 Case Protective Cover             |  Price: $12.99   |  Source: amazon.com
      Protective case compatible with GoPro Hero 12...
  [3] Title: GoPro HERO11 Black Mini                         |  Price: $199.99  |  Source: bestbuy.com

RULES:
1. "Same product" means same TYPE of item:
   - "Disney Rapunzel Perfume" vs "Disney Rapunzel Doll"  → NO (perfume ≠ doll)
   - "Nike Air Force 1" vs "Nike Air Force 1 '07"         → YES (same shoe, variant)
   - "Dyson Airwrap" vs "Dyson V15 Detect"                → NO (different product)
2. Brand must match (or be known sub-brand/parent-brand).
3. Minor naming variations, color/size variants, bundles → YES.
4. Clearly DIFFERENT category of product → NO.
5. Accessories or cases → NO ("iPhone case" ≠ "iPhone").

Output JSON only — one object per result:
[
  {"index": 1, "match": "yes", "reason": "same camera, HERO12 Black is the Hero 12"},
  {"index": 2, "match": "no",  "reason": "this is a protective case, not the camera itself"},
  {"index": 3, "match": "no",  "reason": "Hero11 Mini is a different model"}
]
S06
Seller Authority Filter
Remove products from spam/low-authority sellers using a 306K-entry domain authority table
No LLM ▾

Authority Levels

  • ✅ high — major retailers (keep)
  • ✅ average — established stores (keep)
  • ✅ unknown — not in table (keep, no signal)
  • ❌ low — low-quality stores (filter)
  • ❌ spam — known spam domains (filter)

Lookup Logic

  • Extract domain from search_source
  • Lookup in config/seller_level_authority.tsv
  • 306,868 entries (Region + Domain + Level)
  • Adds seller_authority column to output

Input

  • s05_serp/all_serp_matched.tsv

Output

  • s06_seller/all_seller_ok.tsv
  • s06_seller/all_seller_filtered.tsv
S07
Merge & Final Output
Copy to canonical output path for downstream consumption
No LLM ▾

Input

  • s06_seller/all_seller_ok.tsv

Output

  • s07_merge/merge_all.tsv — final product catalog rows
